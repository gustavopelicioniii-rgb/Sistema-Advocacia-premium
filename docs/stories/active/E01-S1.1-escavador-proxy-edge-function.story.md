# Story E01-S1.1 — Criar Edge Function `escavador-proxy`

**Epic:** E-01 — Segurança: Integrações Externas via Edge Functions
**Status:** Ready
**Prioridade:** 🔴 CRÍTICO

## User Story

Como sistema, quero que chamadas à API Escavador aconteçam no servidor (Edge Function), para que o token da API nunca seja exposto no bundle JavaScript entregue ao browser.

## Contexto Técnico

Atualmente `src/lib/escavador.ts` chama `fetch('https://api.escavador.com/...')` diretamente do browser usando `import.meta.env.VITE_ESCAVADOR_API_TOKEN`. Isso expõe o token no bundle.

A Edge Function `process-monitor-cron` já usa `ESCAVADOR_API_TOKEN` (sem prefix VITE) de forma segura. O padrão já existe — precisamos criar um proxy para chamadas on-demand do usuário.

## Acceptance Criteria

- [ ] Edge Function `escavador-proxy` criada em `supabase/functions/escavador-proxy/index.ts`
- [ ] A função aceita `POST /escavador-proxy` com body `{ processNumber: string, limit?: 50 | 100 | 500 }`
- [ ] A função valida o JWT do usuário via `supabase.auth.getUser()` (apenas autenticados)
- [ ] A função usa `ESCAVADOR_API_TOKEN` do ambiente Supabase (não VITE\_)
- [ ] A função retorna o mesmo formato de `EscavadorMovimentacoesResponse`
- [ ] Em caso de erro da API Escavador, retorna `{ error: string }` com status 4xx/5xx
- [ ] CORS configurado para aceitar apenas a origem do app
- [ ] Sem VITE_ESCAVADOR_API_TOKEN em nenhum arquivo `.env*` após essa story
